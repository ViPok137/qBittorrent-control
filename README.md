# qBittorrent-control

Автоматизированный скрипт для умной очистки и управления загрузками в qBittorrent. Скрипт анализирует активность торрентов и удаляет ненужные, помогая поддерживать порядок в клиенте и экономить место на диске.

## Варианты

| Файл | Платформа | Описание |
|------|-----------|----------|
| `QB_Cleaner_Windows.exe` | Windows | Запускается вручную или через Планировщик задач |
| `QB_Cleaner_Linux` | Linux | Запускается вручную или через cron/systemd |
| `QB_Cleaner_macOS` | macOS | Запускается вручную или через launchd |
| `QB_Cleaner_Server_Windows.exe` | Windows | Серверный вариант — работает непрерывно |
| `QB_Cleaner_Server_Linux` | Linux | Серверный вариант — работает непрерывно |

## Основные возможности

- **Удаление по неактивности**: Автоматическое удаление торрентов старше `X` дней, если их рейтинг (Ratio) не вырос, а количество личей минимально
- **Мгновенная чистка ошибок**: Удаляет торренты со статусами `error`, `missingFiles` или `unknown`
- **Защита от повторной загрузки**: Добавляет удалённые торренты в исключения RSS правил
- **Управление диском**: При заполнении диска выше порога — отключает RSS автозагрузку, паузит или удаляет активные загрузки
- **Автовозобновление**: Когда место освобождается — включает RSS и возобновляет запаузенные торренты
- **Чёрный список**: Торренты с ошибками запоминаются и удаляются при повторном появлении
- **Подробные логи**: Ведёт учёт всех действий в `torrent_log.log`, автоматически удаляет записи старше 30 дней

## Установка и запуск

1. Скачайте готовую версию для вашей ОС из раздела **Releases**
2. Разместите файл в удобном месте и запустите
3. Отредактируйте появившийся `config.ini` под свои настройки
4. Добавьте в автозапуск

### Автозапуск на Windows (Планировщик задач)
1. `Win+R` → `taskschd.msc`
2. Создать задачу → триггер **При входе в систему**
3. Действие → запустить `QB_Cleaner_Windows.exe`

### Автозапуск на Linux (systemd)
```ini
[Unit]
Description=QB Cleaner Server
After=network.target

[Service]
ExecStart=/path/to/QB_Cleaner_Server_Linux
Restart=always

[Install]
WantedBy=multi-user.target
```

## Настройка (config.ini)

При первом запуске создаётся файл конфигурации:
```ini
[SETTINGS]
; Подключение
QB_URL = http://localhost:8080
USERNAME = admin
PASSWORD = adminadmin

; Логика удаления
DAYS_LIMIT = 14
RATIO_THRESHOLD = 1.0
LEECH_SUM_THRESHOLD = 14

; Управление диском
DISK_THRESHOLD_PERCENT = 90
DOWNLOAD_PROGRESS_THRESHOLD = 25
RSS_RULE_NAME = 

; Запуск
STARTUP_WAIT_SEC = 10        ; только для клиентского варианта
CHECK_INTERVAL_HOURS = 1     ; только для серверного варианта
```

### Описание параметров

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `QB_URL` | `http://localhost:8080` | Адрес веб-интерфейса qBittorrent |
| `USERNAME` | `admin` | Логин |
| `PASSWORD` | `adminadmin` | Пароль |
| `DAYS_LIMIT` | `14` | Дней до проверки на удаление |
| `RATIO_THRESHOLD` | `1.0` | Минимальный прирост рейтинга за период |
| `LEECH_SUM_THRESHOLD` | `14` | Минимальная сумма личей за период |
| `DISK_THRESHOLD_PERCENT` | `90` | Порог заполненности диска в % (1-100) |
| `DOWNLOAD_PROGRESS_THRESHOLD` | `25` | Прогресс загрузки в % ниже которого торрент удаляется при забитом диске (1-100) |
| `RSS_RULE_NAME` | _(пусто)_ | Название правила RSS — используется для определения пути сохранения и управления автозагрузкой |
| `STARTUP_WAIT_SEC` | `10` | Пауза перед стартом в секундах (клиентский вариант) |
| `CHECK_INTERVAL_HOURS` | `1` | Интервал между проверками в часах (серверный вариант) |

## Логика работы

### Удаление по неактивности
Каждые 14 дней (настраивается) скрипт проверяет торрент:
- Если прирост рейтинга < `RATIO_THRESHOLD` **И** сумма личей < `LEECH_SUM_THRESHOLD` → удаляет
- Иначе → оставляет и сбрасывает счётчик на новый период

### Управление диском
- Диск заполнен **≥** порога → RSS автозагрузка отключается
  - Загрузки с прогрессом **<** `DOWNLOAD_PROGRESS_THRESHOLD`% → удаляются
  - Загрузки с прогрессом **≥** `DOWNLOAD_PROGRESS_THRESHOLD`% → паузятся
- Диск освободился **<** порога → RSS включается, запаузенные торренты возобновляются
